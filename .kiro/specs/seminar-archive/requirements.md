# 要件定義: ゼミ発表アーカイブWebアプリ

## 概要

大学のゼミにおいて、学生が発表した研究進捗と先生からの指摘（コメント/議事録）を蓄積・検索・閲覧できるアーカイブWebアプリケーション。

## ユーザーロール

| ロール | 説明 |
|--------|------|
| 管理者 (admin) | データのアップロード、プロジェクト管理、ユーザー管理を行う |
| 閲覧者 (viewer) | 発表資料・コメントの検索・閲覧のみ |

## 認証・アクセス制御

- すべてのユーザーに認証を必須とする（Supabase Auth）
- ホワイトリスト方式: 管理者が事前に許可したメールアドレスのみサインアップ/ログイン可能
- 許可メールアドレスリストに含まれないメールでのアクセスは拒否する
- プロジェクト単位のアクセス制御は不要（認証済みユーザーは全プロジェクトを閲覧可能）

## 機能要件

### FR-1: 認証

- FR-1.1: 許可メールアドレスによるサインアップ/ログイン
- FR-1.2: ログアウト
- FR-1.3: 未許可メールアドレスでのサインアップ時にエラー表示

### FR-2: 閲覧機能（全ユーザー）

- FR-2.1: プロジェクト一覧の表示
- FR-2.2: プロジェクト内の発表一覧の表示
- FR-2.3: 発表詳細の表示（中央: コメント/議事録、右サイドバー: 資料）
- FR-2.4: コメント/議事録のスクロール閲覧
- FR-2.5: 資料ファイル（PDF/Word）の表示・ダウンロード
- FR-2.6: 自由文字検索によるフィルタリング
- FR-2.7: カテゴリ付きタグによるフィルタリング（学生名、回生、分野、研究対象、日付）

### FR-3: 管理機能（管理者のみ）

- FR-3.1: 許可メールアドレスの一覧表示
- FR-3.2: 許可メールアドレスの個別追加・削除
- FR-3.3: 許可メールアドレスのCSV一括アップロード
- FR-3.4: プロジェクトの作成・編集・削除
- FR-3.5: 発表の作成・編集・削除
- FR-3.6: 資料ファイルのアップロード（PDF, Word等）
- FR-3.7: コメント/議事録テキストの登録・編集
- FR-3.8: タグカテゴリの作成・編集・削除
- FR-3.9: タグの作成・編集・削除
- FR-3.10: 発表へのタグ付け・タグ解除

## 画面構成

### 共通レイアウト

- ヘッダー: ページ遷移ナビゲーション、アカウント情報（ログアウト）
- 認証必須ページはログインしていない場合リダイレクト

### 閲覧者向け画面

| 画面 | パス（案） | 説明 |
|------|------------|------|
| ログイン | `/login` | メールアドレスによるログイン/サインアップ |
| プロジェクト一覧 | `/projects` | プロジェクトをカード形式で一覧表示 |
| 発表一覧 | `/projects/[id]` | 左サイドバー: フィルター、中央: 発表カード一覧 |
| 発表詳細 | `/presentations/[id]` | 中央: コメント/議事録、右サイドバー: 資料一覧・プレビュー |

### 管理者向け画面

| 画面 | パス（案） | 説明 |
|------|------------|------|
| 管理ダッシュボード | `/admin` | 管理機能のトップ |
| 許可メールアドレス管理 | `/admin/emails` | 一覧、追加、削除、CSV一括アップロード |
| プロジェクト管理 | `/admin/projects` | CRUD |
| 発表管理 | `/admin/projects/[id]/presentations` | CRUD、資料アップロード、コメント登録 |
| タグ管理 | `/admin/tags` | カテゴリ・タグのCRUD |

## 非機能要件

- NFR-1: レスポンシブ対応（PC優先、タブレットは最低限対応）
- NFR-2: 日本語UI
- NFR-3: ファイルアップロードの最大サイズ: 50MB
- NFR-4: Vercelへのデプロイ（フロントエンド）
- NFR-5: Supabaseへのデプロイ（DB, Storage, Auth）

## 技術スタック

| レイヤー | 技術 |
|----------|------|
| フロントエンド | Next.js (App Router) |
| UIライブラリ | shadcn/ui |
| バックエンド | Next.js API Routes |
| データベース | Supabase (PostgreSQL) |
| ファイルストレージ | Supabase Storage |
| 認証 | Supabase Auth |
| デプロイ | Vercel (フロント) + Supabase (バックエンド) |
